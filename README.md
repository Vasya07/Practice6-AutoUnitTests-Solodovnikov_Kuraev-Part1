### 3. Тестирование метода `Debit`
Был написан тест `Debit_WithValidAmount_UpdatesBalance`.

**Результат первого запуска теста (до исправления ошибки):**

**Анализ ошибки:** Тест упал, так как ожидал уменьшения баланса, но метод `Debit` его увеличивал. Это подтвердило наличие логической ошибки в коде.

**Исправление:** В методе `Debit` строка `m_balance += amount;` была заменена на `m_balance -= amount;`.

*   `Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange`

### 5. Рефакторинг кода и тестов
Для повышения информативности код был улучшен:
*   В класс `BankAccount` добавлены константы с сообщениями об ошибках;
*   Конструктор исключения `ArgumentOutOfRangeException` теперь передает значение аргумента и сообщение;
*   Тесты были переписаны с использованием `try-catch` и `StringAssert.Contains` для проверки не только факта исключения, но и правильности его сообщения.

Это позволило точно убедиться, что каждое исключение выбрасывается при правильном условии.

## Вывод
В ходе выполнения практической работы было проведено модульное тестирование класса `BankAccount`.

*   **Причина неуспешного выполнения тестов:** Первоначально тест `Debit_WithValidAmount_UpdatesBalance` не выполнялся из-за логической ошибки в коде метода `Debit` (использование оператора `+=` вместо `-=`).
*   **Причина успешного выполнения тестов:** После исправления ошибки в коде, а также после рефакторинга, все тесты стали успешно проходить. Успешное прохождение тестов `Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange` и `Debit_WhenAmountIsMoreThanBalance_ShouldThrowArgumentOutOfRange` подтверждает, что метод `Debit` корректно обрабатывает ошибочные ситуации и выбрасывает соответствующие исключения с правильными сообщениями. Тесты для метода `Credit` подтвердили его корректную работу.

Таким образом, использование подхода "разработка через тестирование" (в данном случае тестирование уже готового кода) и последующий рефакторинг позволили не только найти и исправить ошибку, но и сделать код более надежным, а тесты — более точными и информативными
